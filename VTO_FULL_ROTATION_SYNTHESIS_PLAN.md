# KORRA: Full-Rotation VTO & View Synthesis Roadmap
**Objective**: Implement a frictionless, multi-angle Virtual Try-On (VTO) experience. The system retrieves existing scan photos (Front/Side), refines them using SAM2, synthesizes a missing "Back View" using the SMPL 3D mesh as a proxy, and executes a synchronized three-angle try-on.

---

## 🏗️ Technical Architecture

### 1. Data Retrieval Layer
*   **Source**: `KORRA_MS.data` (populated in `measurement-screen.js` line 180).
*   **Logic**: Automatically map `photo_front_url` and `photo_side_url` from the scan record.
*   **Friction Removal**: User is only prompted for the **Garment Image**.

### 2. The Refinement Pipeline (Kaggle Backend)
*   **Masking**: Use **SAM2** (implemented in `notebook.ipynb`) to extract the user silhouette from the original scan background.
*   **Normalization**: Image-to-Image pass to convert the user's scan attire into a neutral "gray-skin" base layer.

### 3. View Synthesis (Full-Manifold Wrapping)
*   **Ground Truth**: Use the `.obj` mesh generated by `GarmentRec` (SMPL base).
*   **UV Projection**: Project Refined Front/Side textures (Skin, Hair, and Neutralized Base-Layer) into the SMPL UV-space.
*   **Latent Synthesis**: Rotate camera 180° and use the VLM to "predict" and in-paint the occluded back textures based on the front-facing "Persona Style."
